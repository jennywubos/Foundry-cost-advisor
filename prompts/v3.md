## KEY CONSTRAINTS (sourced from official Microsoft docs)

- **PTU Quota** = per region + per deploy type, **model-independent** (since Aug 2024)
  Source: https://learn.microsoft.com/en-us/azure/foundry/openai/how-to/provisioned-throughput-onboarding
  "Unlike the Tokens Per Minute (TPM) quota used by other Foundry offerings,
   PTUs are model-independent."

- **PTU Capacity (physical availability)** = per region + per model + per deploy type
  Source: Capacity API returns available PTUs per model/region/deployType combination

- **Global Standard actual usage** = global aggregate, cannot be split per region
  (dynamic routing means any request can land in any region)
  Source: https://learn.microsoft.com/en-us/azure/foundry/foundry-models/quotas-limits
  "Global Standard: Deployments share one quota pool across all regions in a subscription."

- **Data Zone Standard actual usage** = per data zone (US or EU), cannot be split per region

- **Regional Standard actual usage** = per region, can be shown directly

- **v3 does NOT show quota** — only actual usage from Azure Monitor API
  Reason: quota numbers without actual usage context are misleading for capacity decisions

- **v3 does NOT calculate costs or prices** — that is v1 and v2's job
  v3's job is capacity visibility and commitment readiness, not billing math

## SECTION 1: COMMITMENT READINESS PANEL

Logic (generated from data, not hardcoded):

```javascript
function assessCommitmentReadiness(region, model, deployType, requestedPTUs) {
  const cap = MOCK_CAPACITY.find(c =>
    c.region === region && c.model === model && c.deployType === deployType);
  if (!cap) return { status:'NO-DATA', color:'#8a8886',
    message:`No capacity data for ${model} in ${region} (${deployType}).` };

  const signal = getDemandSignal(cap.availablePTUs, cap.totalPTUs);
  const canDeploy = cap.availablePTUs >= requestedPTUs;

  // Logic sourced from Microsoft best practice:
  // "Deploy first to confirm capacity, then purchase reservation"
  // Source: https://learn.microsoft.com/en-us/azure/foundry/openai/how-to/provisioned-throughput-onboarding

  if (canDeploy && signal === 'LOW') return {
    status:'GO', color:'#107c10',
    message:`✅ ${cap.availablePTUs} PTUs available in ${region} (${Math.round(cap.availablePTUs/cap.totalPTUs*100)}% free). `
          + `Demand signal: LOW. Safe to commit. `
          + `Recommended: Deploy hourly first to lock capacity → then purchase Annual Reservation.`
  };
  if (canDeploy && signal === 'MEDIUM') return {
    status:'CAUTION', color:'#ca5010',
    message:`⚠️ ${cap.availablePTUs} PTUs available but ${region} demand is MEDIUM (${Math.round(cap.availablePTUs/cap.totalPTUs*100)}% remaining). `
          + `Capacity may tighten. Recommended: Deploy immediately to lock capacity before signing.`
  };
  if (canDeploy && signal === 'HIGH') return {
    status:'CAUTION', color:'#ca5010',
    message:`⚠️ ${cap.availablePTUs} PTUs technically available but demand is HIGH. `
          + `Risk of capacity loss before contract start. Deploy TODAY to lock capacity.`
  };
  if (!canDeploy && signal === 'CRITICAL') return {
    status:'NO-GO', color:'#a4262c',
    message:`❌ ${region} / ${model} is at capacity (${cap.availablePTUs} PTUs remaining, need ${requestedPTUs}). `
          + `Alternatives: ${getAlternativeRegions(model, deployType, requestedPTUs)}. `
          + `Escalate to Azure Capacity Planning for ${region} GPU expansion.`
  };
  return {
    status:'INSUFFICIENT', color:'#a4262c',
    message:`❌ Only ${cap.availablePTUs} PTUs available in ${region}, but ${requestedPTUs} requested. `
          + `Consider splitting across regions or reducing commitment size.`
  };
}

function getAlternativeRegions(model, deployType, minPTUs) {
  return MOCK_CAPACITY
    .filter(c => c.model === model && c.deployType === deployType && c.availablePTUs >= minPTUs)
    .map(c => `${c.region} (${c.availablePTUs} PTUs)`)
    .join(' · ') || 'None currently available';
}
```
